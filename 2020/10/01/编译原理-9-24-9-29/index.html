##1. 文法定义

（由于这里说的“文法”，指的都是“上下文无关文法”，所以简称为“文法”。）

文法自然地描述了大多数程序设计语言构造的层次化语法结构。例如，Java中的if-else语句通常具有以下形式：

if (expression) statement else statement

我们可以发现，一个if-else语句是由关键字if、左括号、一个bool表达式、有括号、一个语句、关键字else、另一个语句组成。

如果我们用“expr”来表示表达式，用“stmt”来表示语句。那么if-else语句的构造方法可以写成如下形式：

stmt -> if (expr) stmt else stmt

其中，箭头（->)可以读作“可以具有以下形式”。
这句语句可以理解为：if-else语句，可以具有以下形式：if关键词、左括号、一个表达式、右括号、一句语句、else关键词、另一句语句。

这样的构造规则，称为“产生式”，

在一个产生式中，像关键字if和左右括号这样的词法元素称为“终结符号(terminal)，因为他们是基本符号，不能被拆分了。

像expr和stmt这样的变量表示终结符号的序列，称为“非终结符号(noterminal)，因为他们可以是一些终结符号的序列。
如一句语句stmt：int a = 5 ; 这里stmt是一些终结符号（int、a、=、5）的序列，所以stmt不能是终结符号，他是非终结符号。

##2. 上下文无关文法

一个上下文无关文法由四个元素组成：

①. 一个终结符号集合，它们有时也被称为“词法单元”。终结符号是该文法所定义的语言的基本符号的集合。通常用小写字母表示，如a、b；
②. 一个非终结符号集合，有时被称为“语法变量”。每个非终结符号表示一个终结符号的集合。通常用大写字母表示，如X、Y；
③. 一个产生式集合。其中，每个产生式包括：一个称为“产生式头”或“左部的非终结符号、一个箭头、一个称为“产生式体”或“右部”的由终结符号或非终结符号组成的序列；
④. 指定一个非终结符号为开始符号。

举例：一个上下文无关文法：
A -> XaY
X -> ab
Y -> a + b
其中，以A为开始符号。

什么是上下文无关？
上下文无关文法就是说这个文法中所有产生式的左边只有一个非终结符。例如：
S -> aSb
S -> a + b
我们只要找到符合产生式右边的符号序列，就可以直接得出该序列对应某个非终结符。
如此例中我们找到了一个序列“a + b”，就可以直接把该序列对应到非终结符S上。

上下文有关文法就是某些产生式左边不止有一个符号，所以匹配该产生式左部的时候还需要考虑这个产生式有正确的上下文。例如：
aSb -> aaSbb
S -> a + b
此例中我们要匹配aSb，必须要找到一个序列，满足左边以a开头，右边以b结尾，中间能匹配到非终结符S的序列。才能匹配这个aSb，所以叫做上下文有关。

##3. 符号表示的约定

①. 小写字母代表终结符号，如a、b、c；
②. 运算符号（+、-）、标点符号、数字0-9、关键词（if、else）代表终结符号；
③. 大写字母代表非终结符号，如X、Y、Z；
④. 字母S通常表示开始符号（非终结符号）；
⑤. 小写、斜体的名字代表非终结符号；
⑥. 把具有相同的产生式头的一组产生式A -> a1，A -> a2，…，A -> ak，写作 A -> a1|a2|…|ak|。其中，a1、a2、……、ak称为A的不同可选体。

##4. 推导

假设一个文法中的非终结符号A，假设文法中存在产生式A -> γ，我们可以推导出αAβ => aγβ，符号 => 表示“通过一步推导出”，在=>符号上加上*号表示“通过零步或多步推导出”，同样的，在=>符号上加+号表示“通过一步或多步推导出”。

如果S =>* α，其中S是文法G的开始符号，我们说α是G的一个“句型”，
注意：一个句型可能既包含终结符号又包含非终结符号，甚至可能是空串。
文法G的一个“句子”是不包含终结符号的句型。

一个文法生成的语言是他的所有句子的集合。

判断：用户写的代码都是句子。（√）

###最左推导和最右推导

假设在一个文法中，
E -> id
E -> E + E

在对E的推导中，E => E + E，我们对右边的哪个E先进行替换呢？

最左推导：总是选择每个句型的最左非终结符进行推导。
本例：E => E + E => id + E => id + id

最右推导同理：
本例：E => E + E => E + id => id + id

###文法二义性

如果一个文法可以为某个句子生成多棵语法分析树，那么它就是二义性的。
二义性文法就是对同一个句子有多个最左推导或多个最右推导的文法。

最左推导或最右推导并不能消除文法二义性！

例子：
stmt -> if (expr) stmt1 if (expr) stmt2 else stmt3

其中，不能知道else是和哪个if匹配的，就会有两种解读，产生二义性。
对于这种情况，我们需要人为的添加规则，如else应和最近的未被匹配的if进行匹配，才可以消除二义性。

##5. 左递归的消除

文法中如果有一个终结符A使得对某个串α存在一个推导 A =>+ Aα，那么这个文法就是左递归的。
自顶向下语法分析方法不能处理左递归的文法，因此需要一个转换方法来消除左递归。

对于一个产生式 A -> Aα | β 处理的一般方法如下：
A -> Aα | β
A -> βA’
A’ -> αA’ | ε

对于一个文法中的以A为起点的产生式：首先将A的全部产生式分组如下：
A -> Aα1 | Aα2 | … | Aαm | β1 | β2 | … | βn

将右侧的A提取公因式：
A -> A(α1 | α2 | … | αm) | (β1 | β2 | … | βn)
就转化成了 A -> AX | Y 的形式。
